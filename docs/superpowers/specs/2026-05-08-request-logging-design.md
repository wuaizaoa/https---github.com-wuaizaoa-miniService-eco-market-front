# 请求日志功能设计文档

**日期：** 2026-05-08  
**作者：** AI Assistant  
**状态：** 待审查

## 1. 概述

### 1.1 目的
为前端应用添加请求/响应日志打印功能，便于调试和问题排查。

### 1.2 范围
- 所有通过 `src/utils/request.ts` 发送的 HTTP 请求
- 包括请求和响应的完整信息
- 所有环境（开发和生产）都打印日志

## 2. 技术方案

### 2.1 技术选型
- 在现有 axios 拦截器中添加日志逻辑
- 使用 `console.group` 进行分组显示
- 过滤敏感信息

### 2.2 模块拆分
- **请求拦截器增强**：在 `request.interceptors.request.use` 中添加请求日志
- **响应拦截器增强**：在 `request.interceptors.response.use` 中添加响应日志（成功和失败）
- **敏感信息过滤工具**：内部辅助函数，用于过滤敏感数据

## 3. 实现思路

### 3.1 请求日志
在请求发送前：
1. 使用 `console.group` 开始分组，标题格式为 `🚀 REQUEST: ${method} ${url}`
2. 打印请求基础信息：URL、Method、Timestamp
3. 打印请求头（过滤敏感信息）
4. 打印请求参数/请求体（过滤敏感信息）

### 3.2 响应日志
在收到响应后：
1. 成功响应：使用 `console.group` 开始分组，标题格式为 `✅ RESPONSE: ${method} ${url}`
2. 失败响应：使用 `console.group` 开始分组，标题格式为 `❌ ERROR: ${method} ${url}`
3. 打印响应状态、响应头、响应数据
4. 使用 `console.groupEnd` 结束分组

### 3.3 敏感信息过滤
- Authorization header：只显示前4个字符，其余用 `****` 代替
- 密码字段：显示为 `***`
- token 字段：显示为 `***`

## 5. 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 生产环境打印大量日志影响性能 | 中 | 确保只打印必要信息，不影响性能 |
| 敏感信息泄露 | 高 | 严格过滤敏感字段 |
| 日志格式不一致 | 低 | 统一日志格式 |

## 6. 验收标准

- [ ] 每次发送请求都能在控制台看到分组的请求日志
- [ ] 每次收到响应都能在控制台看到分组的响应日志
- [ ] 敏感信息（token、密码等）被正确过滤
- [ ] 日志格式清晰易读
- [ ] 不影响现有功能正常运行
